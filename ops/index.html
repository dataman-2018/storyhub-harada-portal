Chapter 2 — Partnership Models

3つのかたちから、
いちばん合うものを。

原田さんの関わり方に合わせて、3つの協業パターンをご用意しました。

Pattern A 共同事業開発型
報酬
レベニューシェア（固定費なし）
関与
PJ業務に直接参画
メリット
お互いが成果に本気でコミットできる形です。StoryHub側のキャッシュ負担もありません
デメリット
案件が決まるまで、原田さん側に収入が発生しない点はご留意ください
適するケース
一緒に顧客を開拓し、事業開発に深く関わっていく場合に向いています
Pattern B ブランド顧問型
報酬
月額固定
関与
定例 + 壁打ち（PJ業務には直接関与しない）
メリット
安定的に知見をお借りできます。ご負担も軽めです
デメリット
成果との結びつきがやや薄くなります
適するケース
原田さんの稼働が限られる場合に向いています
Pattern C ハイブリッド型
報酬
月額ベース（低め）+ 案件参画時にシェア上乗せ
関与
基本は壁打ち、案件時に参画
メリット
安定した関係をベースに、案件ごとの上乗せがあります
デメリット
条件の設計がやや複雑になります
適するケース
長いお付き合いを前提に、段階的に関わりを深めていく場合に向いています
初期フェーズではパターンA または C がおすすめです。
壁打ちだけでは、本当の価値は生まれにくいと考えています。原田さんが顧客と直接向き合ってこそ、ソリューションの品質が決まります。
Chapter 3 — Working Style

日常の動き方。

定例ミーティング
  • 週1回 または 隔週（まずはすり合わせて決めましょう）
  • 事業開発の進捗共有、ソリューション設計のディスカッション、ブランドまわりの壁打ち
プロジェクト業務（クライアント案件時）
  • ブランドコンセプト策定ワークショップの設計・ファシリテーション
  • ブランドメモリーの要件定義（攻めのトーン＋ガードレール）
  • クライアントへのプレゼンテーション・レビュー
コンテンツ発信
  • Podcast「企業版古典ラジオ」での対談・ブランド哲学の発信
  • StoryHub × 原田さんの記事コンテンツ
  • 将来的に書籍化（「ストーリー経営」概念の共同定義）
稼働の記録と見直し
  • 稼働時間はお互いに記録し、3ヶ月ごとに振り返ります
  • 案件の増減に応じて、柔軟に調整していきましょう
